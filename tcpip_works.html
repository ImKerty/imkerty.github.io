FIRST STEP - Data Segmentation:

When a device wants to send data, the data is divided into smaller chunks called packets. Each packet is assigned a sequence number.

SECOND STEP - Addressing (IP Layer):

Each packet is tagged with the IP addresses of the source and destination devices, allowing routers to route the packets across the Internet.

THIRD STEP - Routing (Internet Layer):

Routers use the destination IP address to determine the best path for the packets to reach their destination, involving multiple hops across networks.

FOURTH STEP - Transmission Control (TCP Layer):

If reliability is crucial, TCP establishes a connection between the source and destination devices, ensuring data is delivered without errors and in the correct order.

FITFH STEP - End-to-End Communication:

Once the packets arrive at the destination, they are reassembled into the original data. If TCP is used, the receiving device sends acknowledgments back to the sender.

SIXTH STEP - Application Layer:

The final step involves delivering the reassembled data to the appropriate application, managed by protocols like HTTP for web browsing or FTP for file transfers.
